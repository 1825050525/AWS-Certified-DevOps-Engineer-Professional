## 戦略の核心：「弱点の潰し込み」に全集中

問題集3周完了は十分な準備。この段階でやるべきことは：

```
❌ 新しいサービスを一から学ぶ
❌ テキストを読み直す
✅ 間違えた問題を徹底的に潰す
✅ 混同しやすいサービスの差異を整理する
✅ 本番の時間配分に慣れる
```

## 残り11日 日別プラン

### Day 1〜3（4/30〜5/2）：弱点洗い出し
- Udemy問題集を1周して間違えた問題を全て `02_weak_points/weak_points.md` に記録
- ドメイン別の正答率を計算して優先ドメインを確定
- **目標：弱点問題を全てリスト化する**

### Day 4〜7（5/3〜5/6）：弱点集中補強
- 弱点リストの問題を1問ずつ「なぜ間違えたか」を言語化
- チューターエージェントで混同しやすいサービスを整理
- Black Beltで苦手サービスだけ確認（動画全体は不要・ポイントだけ）
- **目標：弱点問題の正答率80%以上**

### Day 8〜9（5/7〜5/8）：模擬試験モード
- Udemy問題集を時間計測で通し解き（180分）
- 時間が余る場合の見直し戦略を確認
- **目標：模擬試験スコア750点以上（75%換算）**

### Day 10（5/9）：前日
- 弱点リストを軽く見直すだけ（新しいことは一切やらない）
- 試験会場・時間・持ち物確認
- 早めに就寝

### Day 11（5/10）：試験当日
- 試験2時間前に起床・軽食
- 弱点リストを10分だけ眺める
- **受験・合格🎉**

## ドメイン別 優先度と戦略

### 🔴 最優先：ドメイン1（SDLC 22%）
配点最大。CodeDeploy のデプロイ戦略・CodePipeline の構成が最頻出。
- Blue/Green vs Canary vs Rolling vs In-Place の使い分け
- CodeDeploy の AppSpec ファイル
- ECS/Lambda へのデプロイ方法の違い

### 🔴 重要：ドメイン6（セキュリティ 17%）
SCPとIAMポリシーの組み合わせ問題が増加傾向。
- SCP（許可の上限設定）vs IAMポリシー（実際の権限設定）の違い
- Permission Boundary の動作
- Secrets Manager vs Parameter Store（SecureString）の使い分け

### 🟡 重要：ドメイン2（IaC 17%）
CloudFormationの細かい挙動が出る。
- CloudFormation ドリフト検出・変更セット・スタックセット
- SSM Parameter Store を CFn で参照する方法
- CDK vs SAM の違い・使い分け

### 🟡 注意：ドメイン3（耐障害性 15%）
DRパターンのRTO/RPOを絡めた問題が出る。
- RTO/RPO × DRパターンの対応表
- Route 53 フェイルオーバールーティング
- Aurora Global Database vs DynamoDB Global Tables

### 🟢 安定：ドメイン4・5（モニタリング・インシデント 各15%・14%）
CloudWatch / EventBridge は実務経験があれば解けるものが多い。
弱点があれば補強。なければ時間を割かない。

## 本番の時間配分戦略

```
75問・180分 → 1問あたり2分24秒

推奨フロー：
0〜90分  ：全問を1周（わからない問題はフラグしてスキップ）
90〜150分 ：フラグ問題を再検討
150〜180分：見直し・最終確認

フラグの基準：
・30秒考えても選択肢が2つに絞れない → フラグしてスキップ
・複数選択で「何問選ぶか」がわからない → フラグしてスキップ
```

---

## よく出る引っかけパターン

### パターン1：「最も運用負荷が低い」
→ フルマネージドサービスを選ぶ（EC2よりFargate、RDSよりAurora Serverless）

### パターン2：「既存のコードを変更せずに」
→ Systems Manager / Lambda / EventBridge での自動化を選ぶ

### パターン3：「マルチアカウント環境で」
→ Organizations + SCP / CloudFormation StackSets / AWS Config 組織ルールを選ぶ

### パターン4：「コスト効率よく」
→ Reserved Instance・Savings Plans・スポット活用を疑う

### パターン5：「セキュリティを最大化」
→ 最小権限の原則 / Secrets Manager（ハードコード禁止）/ KMS暗号化

## 混同しやすいサービス対比表

| 比較 | A | B | 使い分け |
|------|---|---|---------|
| デプロイ | CodeDeploy In-Place | Blue/Green | 既存インフラ再利用 vs 新環境に切替 |
| 設定管理 | Parameter Store | Secrets Manager | 設定値 vs 機密情報（自動ローテーション） |
| イベント | EventBridge | SNS | ルーティング複雑 vs シンプル通知 |
| コンテナ | ECS | EKS | AWS管理 vs Kubernetes互換 |
| IaC | CloudFormation | CDK | 宣言的YAML vs プログラミング言語で記述 |
| DR | Pilot Light | Warm Standby | 最小限起動 vs 縮小スケールで常時稼働 |
