# 試験テクニック・ノウハウ - DOP-C02

## 試験の特徴（DOP特有）
- 問題文・選択肢が非常に長い（SAA・SOAより明らかに長い）
- 複数サービスの**組み合わせ**で答えるパターンが多い
- 「最も〇〇な方法」という優劣を問う問題が多い
- 採点対象外が10問あるため、わからなくても諦めない

## 時間配分（180分・75問）
```
1問あたり：2分24秒

推奨フロー：
前半 0〜90分  → 全問1周（迷ったらフラグ・スキップ）
後半 90〜150分 → フラグ問題を再検討
最後 150〜180分 → 見直し・空欄確認
```

## 問題を解く順番の戦略
1. 問題文を最後まで読んでから選択肢を見る
2. 「2択」まで絞ってから再読する
3. 30秒で絞れない → フラグしてスキップ
4. 後半にまとめてフラグ問題を処理

## 選択肢の絞り方

### 「運用負荷が最小」「自動化」が問われたら
→ マネージドサービス優先（Fargate > EC2、RDS > 自前DB）
→ Systems Manager / Lambda / EventBridge での自動化

### 「マルチアカウント」が条件にある
→ Organizations + SCP / CloudFormation StackSets / AWS Config 組織ルール

### 「セキュリティ強化」が条件にある
→ Secrets Manager（ハードコード禁止）/ KMS / IAM最小権限 / VPC エンドポイント

### 「障害時の自動復旧」
→ EventBridge + Lambda / Systems Manager Automation / Auto Scaling

### 「コスト効率」
→ Lambda（使った分だけ）/ Spot / Savings Plans / S3 Intelligent-Tiering

## 複数選択問題のコツ
- 問題文に「2つ選択」「3つ選択」と明記されている
- 必ず指定数を選ぶ（多くても少なくてもダメ）
- 「明らかに違う選択肢」を先に除外して絞る

## 本番での言語設定
- 日本語で受験しつつ、わかりにくい問題は英語に切替可能
- 試験中いつでも言語を切替できる
